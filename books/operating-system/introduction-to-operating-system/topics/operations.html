Q1. Why is the distinction between user mode and kernel mode fundamental to operating system security?

🔹 A. It allows the OS to run faster by ignoring user requests.
🔹 B. It prevents user programs from executing privileged instructions that could compromise system integrity.
🔹 C. It enables the CPU to switch between different physical processors.
🔹 D. It allows multiple users to share the same memory space without isolation.

💡 Difficulty: easy | ✅ Correct: B

Q2. If a user program attempts to execute a privileged instruction, such as disabling interrupts, what is the immediate hardware response?

🔹 A. The instruction is executed silently.
🔹 B. The CPU switches to a higher privilege level automatically.
🔹 C. The hardware treats it as an illegal instruction and traps to the operating system.
🔹 D. The instruction is ignored and the program continues.

💡 Difficulty: medium | ✅ Correct: C

Q3. Consider a scenario where an operating system runs entirely in user mode. What is the primary vulnerability of this architecture?

🔹 A. It would be unable to perform any I/O operations.
🔹 B. It would lack the hardware enforcement mechanism to prevent user processes from modifying critical system data structures.
🔹 C. It would require significantly more power.
🔹 D. It would not be able to support multitasking.

💡 Difficulty: hard | ✅ Correct: B
